# AGENTS.md — Semar Kos Finder

## Architecture

Three apps + MongoDB:

| Dir | Stack | Port (dev) | Notes |
|-----|-------|-----------|-------|
| `frontend/` | Next.js 16, React 19, MapLibre GL | 3000 | Public map, fetches `/api/kos` from backend |
| `admin/` | Next.js **14**, React 18 | 3001 | Admin dashboard, CRUD, LLM data cleaning |
| `backend/` | FastAPI (Python 3.11), Motor (MongoDB async) | 8000 | REST API, JWT auth, job queue |
| — | MongoDB 7 | — | Shared across all profiles |

`src/` is standalone Python data-processing scripts (not part of backend service). Run with host Python/uv.

## Package managers

| Directory | Tool | Command |
|-----------|------|---------|
| `frontend/` | npm | `npm run dev`, `npm run build` |
| `admin/` | npm | `npm run dev`, `npm run build` |
| `backend/` | **uv** | `uv run pytest`, `uv run python -m app.seed` |
| `src/` | uv (or bare Python) | `uv run python src/csv_to_json.py` |

**Never use `pip` or bare `python` for backend.** Always `uv run`.

## Running the stack

### Development (Docker, hot reload)
```bash
cp .env.development.example .env.development
# Edit .env.development — fill GOOGLE_MAPS_API_KEY, JWT_SECRET, ADMIN_PASSWORD_BCRYPT

docker compose --profile development up -d --build
docker compose --profile development run --rm backend_dev uv run python -m app.seed
```

### Frontend only (no Docker)
```bash
cd frontend && npm run dev   # → http://localhost:3000
```
Requires backend running separately.

### Production / Staging
Never start manually. Deployed via CI:
- Push to `main` → GitHub Actions → production deploy
- Push to `staging` → GitHub Actions → staging deploy
Both workflows use **GitHub Environments** for secret isolation.

## Testing

### Backend (pytest)
```bash
cd backend && uv run pytest          # all tests
cd backend && uv run pytest -k "test_health_ok"  # single test
```
Tests mock MongoDB (`monkeypatch.setenv` + `AsyncMock`). No real DB needed.

### Admin (Playwright e2e)
```bash
# Via Docker:
docker compose --profile development run --rm admin_e2e
```
Tests in `admin/e2e/`. Base URL: `http://admin_dev:3001`.

### Frontend
No tests configured.

## Environment files

| File | Used by |
|------|--------|
| `.env` | `src/` scripts (root Python) |
| `.env.development` | Docker dev profile |
| `.env.production` | Docker prod profile |
| `.env.staging` | Docker staging profile |

All `*.env*` except `.example` are gitignored. Key env vars: `GOOGLE_MAPS_API_KEY`, `MONGO_URL`, `JWT_SECRET`, `ADMIN_PASSWORD_BCRYPT`, `LLM_API_KEY` (optional).

## Data quirks

- **Dataset is close-source.** `data/`, `dump/`, `notebooks/`, `notes/` are all gitignored.
- `frontend/public/data/` is gitignored — generated by `src/csv_to_json.py`.
- The `.ignore` file tells ripgrep to search `frontend/public/data/` (negates gitignore).
- Data pipeline: `src/extract_pdf.py` → `notebooks/cleaning.ipynb` → `src/geocoding_location.py` → `src/csv_to_json.py` → frontend.

## Backend details

- Entrypoint: `backend/app/main.py:create_app()`
- Config loaded via `pydantic-settings` from env vars, accessed via `load_config()` in `config.py`.
- JWT auth in `app/auth.py`. Admin routes under `/api/admin/*`.
- LLM data cleaning: `app/parse_engine.py` + job queue in `app/job_queue.py`. Admin UI routes in `app/routers/admin_actions.py`.
- `ss` collection for master location data (UNS buildings), `kos` for cleaned kos data.

## CI/CD

Two workflows in `.github/workflows/`:
- `production.yml` — triggers on push to `main`
- `staging.yml` — triggers on push to `staging`

Both build/push Docker images to `ghcr.io/wildanjr19/semar-kos-*`, then SSH deploy to separate VPS servers. Secrets read from GitHub Environments (not repo secrets).

## Linting / typechecking

- **Frontend/Admin**: No lint or typecheck scripts in `package.json`. TypeScript `strict: false`.
- **Backend**: `pyrightconfig.json` at repo root with venv pointing to `.venv`. Run: `pyright backend/` or `uv run pyright backend/`.

## Commit conventions

- No specific branch naming or commit message convention documented.
- `main` = production, `staging` = staging. Feature branches merge to staging first.
- Never commit `data/`, `.env*`, or `skills-lock.json`.
